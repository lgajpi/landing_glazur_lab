# Glazur Lab — лендинг

Продающий одностраничник: цветы и украшения ручной работы из ювелирной смолы и
полимерной глины. Вечная красота для интерьера и подарков — без ухода.

## Файлы

| Файл | Что внутри |
|------|-----------|
| `index.html` | разметка всех блоков |
| `styles.css` | все стили, анимации, адаптив |
| `script.js` | каталог работ, карусель, анимация «рождения», форма |
| `images/` | фото работ (используются в карусели и украшениях) |

Открывается двойным кликом по `index.html`. После правок в браузере —
**жёсткое обновление `Ctrl + Shift + R`** (иначе тянется старый CSS/JS из кеша).

## Блоки страницы (сверху вниз)

1. **Hero** — заголовок, подзаголовок, кнопки, плавающие лепестки.
2. **Как рождается цветок** — SVG-анимация из 5 фаз: каркас → лепестки →
   погружение → **УФ-закалка** (по контуру бежит светящийся «пиксель со шлейфом») →
   роспись.
3. **Каталог работ** — карусель: крупное фото, фон секции плавно перетекает
   (кроссфейд) в цвет каждой работы. Сначала видно чистое фото, через ~2,2 сек
   разом всплывают название + описание + цена.
4. **Украшения** — броши, серьги (реальные фото), кулоны/заколки.
5. **Почему это особенное** — 3 аргумента (перед формой заказа).
6. **Заказать + контакты** — форма заявки со светящейся бегущей рамкой + ВКонтакте.
   На мобилке порядок: текст → форма → контакты.

## Каталог и цены — где править

Всё в одном месте — массив `flowers` (и `jewelry`) в начале `script.js`.
Каталог питает и карусель, и выпадающий список в форме заказа.

Чтобы задать цену — поменять `price: null` на строку, например:

```js
{
  name: 'Красная роза',
  file: 'красная роза.jpg',
  desc: 'Глянцевый бордо с золотой каймой лепестков.',
  price: '3 500 ₽',   // ← было null → показывалось «Цена по запросу»
  grad: ['#4a0a12', '#8f1424', '#c8324a'],
  dark: true,
},
```

Работы в каталоге сейчас (9 цветов + 2 украшения):

- Красная роза
- Сирень
- Голубая гортензия
- Розовая гортензия
- Космеи
- Белая лилия
- Большие подснежники
- Мелкие подснежники
- Ромашки
- Брошь из гортензии *(jewelry)*
- Серьги-цветок *(jewelry)*

### Добавить новую работу
1. Положить фото в `images/`.
2. Добавить объект в массив `flowers` (или `jewelry`) в `script.js`:
   `name`, `file` (имя файла), `desc`, `price`, `grad` (3 цвета фона), `dark`
   (`true` для тёмного фото — текст белый, `false` для светлого).

## Настройки поведения (в `script.js`)

- `PRICE_DELAY` — пауза перед показом подписи и цены (сейчас `2200` мс).
- `PRICE_FALLBACK` — текст, когда цена не задана (`'Цена по запросу'`).
- автопрокрутка карусели — `setInterval(next, 6500)`.

## Заметки по правкам

- УФ-свечение **не** продаётся как фишка продукта (не все работы светятся) —
  ультрафиолет упомянут только как этап производства (закалка глины).
- Бегущий «пиксель со шлейфом» есть в двух местах: по контуру цветка в анимации
  (фаза УФ-закалки) и по рамке формы заказа (подсветка CTA).
- Поле «Что хотите заказать» — не нативный `<select>`, а кастомный список
  (`.wsel` в разметке). При наведении на пункт над ним всплывает плавающий попап
  (`.wsel__pop`, живёт на `<body>`, позиционируется через JS) с фото работы.
  Значение уходит в скрытый `input[name="wish"]`. Список строится из `flowers` +
  `jewelry`, поэтому у украшений в массиве тоже указан `file` (фото для превью).
